== Pre-flight checklist

Run these before starting the demo:

[source,bash,subs="attributes+"]
----
cd rbac-demo/
./scripts/deploy.sh
./scripts/validate.sh   # must exit 0 before starting
----

Then open the web console and navigate to: *Administrator -> User Management -> RoleBindings*

Confirm workloads are running:

[source,bash,subs="attributes+"]
----
oc get pods -n nf-control-plane
oc get pods -n nf-user-plane
oc get pods -n observability
----

== Troubleshooting common issues

*OAuth not applied / users cannot log in*

The OAuth patch in `scripts/deploy.sh` requires a cluster restart of the OAuth server. If users see "Invalid credentials," run:

[source,bash,subs="attributes+"]
----
oc rollout restart deployment/oauth-openshift -n openshift-authentication
oc rollout status deployment/oauth-openshift -n openshift-authentication
----

Then wait 60 seconds before retrying login.

*`validate.sh` fails with missing bindings*

The Kustomize deployment must complete fully before bindings resolve. Re-run:

[source,bash,subs="attributes+"]
----
oc apply -k install/
sleep 30
./scripts/validate.sh
----

*Workloads not running (ImagePullBackOff)*

The demo uses UBI images from `registry.access.redhat.com`. Confirm the cluster has outbound internet access, or pre-pull and mirror the images to an internal registry.

*`oc adm node-logs` returns no output (Scenario 10)*

Audit logging is enabled by default on OCP 4.16+. If the command returns nothing, wait 60 seconds after running some `oc` commands as demo users to generate log entries, then retry.

*`self-provisioners` patch cannot be undone*

The cleanup script restores `self-provisioners` from the backup. If Scenario 7 fails mid-way, restore manually:

[source,bash,subs="attributes+"]
----
oc patch clusterrolebinding self-provisioners \
  --type=json \
  -p '[{"op":"replace","path":"/subjects","value":[{"apiGroup":"rbac.authorization.k8s.io","kind":"Group","name":"system:authenticated:oauth"}]}]'
----
